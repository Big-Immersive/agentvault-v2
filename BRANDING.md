# AgentVault Brand Identity

---

## Logo Concept

**Primary Mark: Shield + Vault Door**
A geometric shield shape with a vault door mechanism (circular dial with spokes) integrated into its center. The shield represents protection; the vault door represents controlled access. The dial has 3 notch marks at different positions representing trust levels (restrictive, moderate, permissive).

**Construction:**
- Outer shape: Angular shield with slightly rounded bottom point
- Inner element: Circular vault dial with 6 spokes radiating from center
- Accent: Small key-lock icon at the 12 o'clock position on the dial
- Optional: Subtle circuit-board lines running through the shield's surface (tech/AI motif)

**Monogram:** "AV" with the A forming a shield shape and the V nested inside as a downward-pointing lock mechanism.

**Favicon:** Simplified vault dial inside shield silhouette, works at 16x16px.

## Brand Voice & Tone

### Voice
- **Authoritative but approachable** — We know security, but we don't gatekeep
- **Developer-native** — Speak in CLI commands, not marketing fluff
- **Direct** — Say what it does. No jargon padding.
- **Slightly urgent** — The problem is real and happening now

### Tone Guidelines
- Lead with the problem, follow with the solution
- Use concrete examples (real env var names, real attack scenarios)
- Humor is OK but dry/technical (not playful/cute)
- Never condescend about security practices
- Terminal output > paragraphs of explanation

### Writing Examples
- ✅ "Your AI agent can see your AWS keys. AgentVault stops that."
- ✅ "Three commands. Full credential control."
- ❌ "Leverage our cutting-edge AI security paradigm"
- ❌ "Don't worry, we've got you covered! 🛡️"

## Visual Style Direction

### Overall Aesthetic
Dark, technical, premium. Think: Linear meets 1Password meets a terminal window. Security product that developers actually want to use.

### Key Visual Elements
- **Terminal windows** — Real CLI output as hero imagery (not stock photos)
- **Glow effects** — Subtle blue glow on interactive elements (shield motif)
- **Grid backgrounds** — Faint dot grid or line grid on dark backgrounds
- **Color-coded access levels** — Green (allowed), Amber (redacted), Red (denied) consistently throughout
- **Glassmorphism cards** — Frosted glass effect on dark backgrounds for feature cards
- **Code blocks** — Syntax-highlighted terminal output as primary visual content

### What to Avoid
- Generic shield/lock stock icons
- Bright/playful color schemes
- Rounded/bubbly UI elements
- Stock photography of any kind
- Gradients that look like 2018 (purple→pink)

### Inspiration References
- Linear.app — dark theme, clean typography, premium feel
- Vercel — developer-focused, terminal-centric marketing
- Warp.dev — terminal product branding done right
- Snyk — security product with developer-friendly positioning
